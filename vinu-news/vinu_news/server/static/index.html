    Database Tables Information

    The system runs on a SQLite database containing 10 logical tables (plus FTS5 search index tables). Here is a breakdown of what each table is used for:

    1. Ingestion & Content

    • articles: Stores processed news leads (headline, summary, source, sentiment, impact, and other extraction metadata).
    • watchlist_tickers: Contains active stock symbols being monitored (e.g. AAPL, NVDA).
    • vinu_settings: Stores runtime application config (e.g. collection mode, poll interval).
    • feed_health: Monitors feed latency, success rates, failure streaks, and recent HTTP/parse errors for the 22 RSS channels.

    2. Grouping & Analytics

    • story_threads: Groups semantically similar articles together over time into unified story lines.
    • article_ticker_mentions: A junction table mapping articles to tickers with dominance weights and primary indicators.
    • thread_daily_snapshots: Stores daily aggregated statistics (sentiment distributions) per story thread.
    • ticker_daily_stats: Aggregates daily metrics per ticker (total articles, bullish/bearish count).

    3. Enrichment & Integrations

    • news_analysis: Caches the on-demand, deep LLM analysis (sentiment, confidence score, risk flags, and summary).
    • article_price_reaction: Tracks price differences before and after an article is published for primary tickers (integrates with stock service).

    ⚙️ Ingestion & Analysis Pipeline (Without LLM)

    Deterministic rule-based ingestion. Every poll cycle runs this pipeline to process feed content quickly.

    1. Ingestion Source Loads RSS feeds configured in feeds.yaml and queries Yahoo News via run_ticker_news_ingest.
    2. Pre-Enrichment & 9-Stage Rules Filters batch duplicate URLs, strips HTML, and classifies:
    • Priority: FLASH / URGENT / BREAKING / ROUTINE
    • Sentiment: BULLISH / BEARISH / NEUTRAL
    • Impact: HIGH / MEDIUM / LOW based on rules
    • Category: EARNINGS, CRYPTO, DEFENSE, TECH, etc.
    • Ticker Extraction: C++ 20-word stop list filters symbols
    • Threat & Source Credibility: Evaluates site and content risk indicators
    3. Post-Enrichment Deduplication & Lead Pick Performs Country & Person NER. Normalizes synonyms, vectorizes headlines (TF-IDF), clusters duplicates with cosine similarity ≥ 0.25, and picks the best "Lead" article to save.
    4. Persistence & Thread Matching Locks and commits raw news leads to the SQLite database (updating articles, FTS index, daily snapshots, and matches to existing story_threads).

    🧠 Deep Article Analysis Pipeline (With LLM)

    On-demand intelligence trigger. Only runs when a client asks to analyze an article's context.

    1. API Request Trigger Client sends a POST /news/analyze containing the article ID or link. The system looks up the article in the raw articles table.
    2. Cache Table Lookup The system queries the news_analysis cache table. If a match is found and its age is within the configured TTL (e.g. 24 hours), the cached JSON result is returned immediately.
    3. LLM API Query & Cache Store On a cache miss, the system formats the user prompt (injecting article details), queries the OpenAI-compatible or local Ollama (llama3.2) service, parses and standardizes the JSON response, stores it in news_analysis, and returns it.
